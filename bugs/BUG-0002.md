# BUG-0002: 肌専用ディザがモードOFF時に無視される
- 報告日: 2026-03-20
- ステータス: 完了
- 重要度: 中
- 発見方法: Playwright自動テスト（T-06）

## 現象
肌感改善モードを「OFF」にした状態で肌専用ディザをONにして変換実行しても、ディザが適用されない。変換結果がディザOFF時と完全一致する。

## 原因
`app.js` の `getSkinOptions()` が `skinMode === 'off'` の場合に即座に `return null` していたため、肌専用ディザの有効状態を確認せずに `skinOptions` 自体が送信されなかった。

```javascript
// 修正前
function getSkinOptions() {
    const skinMode = document.getElementById('skin-mode').value;
    if (skinMode === 'off') return null;  // ← ここでディザ設定も巻き込んでnull
    ...
}
```

## 処置内容
モードOFFかつディザも無効の場合のみ `null` を返すよう条件を変更。

```javascript
// 修正後
function getSkinOptions() {
    const skinMode = document.getElementById('skin-mode').value;
    const skinDitherEnable = document.getElementById('skin-dither-enable').checked;
    if (skinMode === 'off' && !skinDitherEnable) return null;
    ...
}
```

変更ファイル:
- `src/js/app.js` — `getSkinOptions()` の早期リターン条件を修正

## 再発防止策
肌感改善パネルのモード選択（A/B排他）と肌専用ディザは独立した機能であり、SPECにも「機能Bが OFF の場合でも、肌専用ディザ用にマスクは内部生成する」と明記されている。UIの状態を `skinOptions` に集約する際、独立した機能同士を一つの条件で判定しないこと。
