# BUG-0003: MAGエンコーダが不正なファイルを出力する
- 報告日: 2026-03-21
- ステータス: 完了
- 重要度: 高
- 発見方法: 98エミュレータでの表示確認

## 現象
MAG形式で保存したファイルが98エミュレータで正しく表示されない。
- NG1: デコードエラー（ファイル構造が不正）
- NG2: 全面黒画像（パレットが壊れている）

## 原因（3件）

### 原因1: デルタテーブルの誤り
ネット上の断片情報からデルタテーブルを構築したが、フラグ9-13の位置が間違っていた。
MAGBIBLE.DOCの図を正しく読み取ると、右方向のデルタ(+1)は存在しない。

```
誤: 9=(+1,-1), 10=(+1,-2), 11=(-2,-4), 12=(-1,-4), 13=(0,-4)
正: 9=(0,-4),  10=(-1,-4), 11=(-2,-4), 12=(0,-8),  13=(-1,-8)
```

### 原因2: ヘッダのサイズフィールドにオフセット値をコピーしていた
FlagBサイズとPixelサイズのフィールドに、サイズではなくオフセット値を書き込んでいた。

### 原因3: パレットの格納形式の誤り
4bit値(0-15)をバイトの下位ニブルにそのまま格納していたが、
MAGBIBLE.DOCの仕様では上位ニブルに格納し下位ビットを1で埋める必要がある。

```
誤: 値4 → 0x04（ビューアは上位4bitを読むので0として解釈→黒）
正: 値4 → 0x4F（上位4bit=4、下位は1で埋める）
```

## 処置内容
`mag-export.js` を全面書き直し。MAGBIBLE.DOC準拠の実装に変更。

1. デルタテーブル修正（全16位置を仕様書の図から正確に読み取り）
2. ヘッダのサイズフィールドを実サイズに修正
3. パレット格納を `expand4to8()` で上位ニブル格納に修正
4. 1ピクセル=4ドット(2バイト)の正しいMAGピクセル単位に変更
5. フラグ圧縮を仕様通りの縦XOR＋FlagA/B分離に変更
6. 偶数バンダリ調整を追加

変更ファイル:
- `src/js/io/mag-export.js` — 全面書き直し

## 再発防止策
**一次資料を最優先で参照すること。** 今回はMAGBIBLE.DOC（公式仕様書）がdocs/に配置されていたにもかかわらず、ネット上の断片的な二次情報から実装してしまった。フォーマット実装時は必ず一次仕様書を読み、不明点がある場合は既知の正常ファイルとのバイナリ比較で検証する。
