# GAZO98 追加仕様: 肌感改善エンジン

## SKIN_ENHANCEMENT v1.0

本ドキュメントはGAZO98本体SPEC.mdに対する追加仕様であり、Phase 3（パレット＆減色エンジン）およびPhase 4（ディザリングエンジン）に機能追加を行う。

---

## 1. 課題分析

### 1.1 現状の問題

現行のメディアンカット法は画像全体の色分布を均等に分割するため、以下の問題が発生する:

- 背景（空、室内壁など広い面積の色）にパレット枠を多く消費する
- 肌色領域に割り当てられるパレットが2〜3色に留まる
- 肌のグラデーションが急峻になり、ディザリングでも補いきれない
- 結果として「汚い」「ノイジー」な肌表現になる

### 1.2 98ドット絵師のアプローチ（参考）

当時のプロは16色中6〜8色を肌色系に割り当て、以下のような階調を確保していた:

```
色番号  用途               典型的なRGB(4bit)
------  ----               -----------------
  0     最暗影（輪郭寄り）   R:5  G:3  B:2
  1     暗部の影             R:8  G:4  B:3
  2     影                   R:10 G:6  B:4
  3     中間色               R:12 G:8  B:6
  4     標準肌色             R:13 G:10 B:8
  5     明部                 R:14 G:11 B:9
  6     ハイライト           R:15 G:13 B:11
  7     最明部（白寄り）     R:15 G:14 B:13
```

この「肌色優先配分」をアルゴリズムで再現するのが本仕様の目的である。

## 3. 機能A: 肌色優先パレット

### 3.1 概要

メディアンカットの色空間分割時に、肌色領域に対して重み付けを行い、パレット枠を優先的に配分する。

### 3.2 肌色検出

HSV色空間で以下の条件を満たすピクセルを「肌色候補」として検出:

```
肌色判定条件（HSV空間）:
  H (色相):     0° ≤ H ≤ 50°  （赤〜オレンジ〜黄色）
  S (彩度):     0.1 ≤ S ≤ 0.7  （薄すぎず濃すぎず）
  V (明度):     0.2 ≤ V ≤ 0.95 （暗すぎず白飛びせず）
```

追加条件（RGB空間での補助フィルタ）:
```
  R > G > B （赤が最大、青が最小）
  R - G ≤ 80  （赤と緑の差が大きすぎない）
  R - B ≥ 20  （赤と青にある程度の差がある）
```

### 3.3 重み付きメディアンカット

1. 全ピクセルに対して肌色判定を実行
2. 肌色と判定されたピクセルの出現頻度に **重み係数** を乗算
3. 重み係数はUIスライダーで調整可能

```
パラメータ:
  skin_weight: 1.0〜5.0（デフォルト: 3.0）

  重み適用:
    肌色ピクセルの出現カウント × skin_weight
    非肌色ピクセルの出現カウント × 1.0
```

4. 重み付きヒストグラムに対してメディアンカットを実行
5. 結果として肌色領域に多くの分割（＝多くのパレット枠）が割り当てられる

### 3.4 パレット枠の最低保証

skin_weightだけでは不十分な場合に備え、肌色用パレット枠の最低数を保証する:

```
パラメータ:
  min_skin_colors: 0〜8（デフォルト: 4）

処理:
  1. メディアンカット実行後、肌色に分類されたパレット数をカウント
  2. min_skin_colors未満の場合:
     a. 非肌色パレットの中で使用頻度が最も低いものを除去
     b. 肌色領域を追加分割して新たな肌色パレットを生成
     c. min_skin_colorsを満たすまで繰り返す
```

### 3.5 UI追加

パラメータパネルに以下を追加:

```
┌─ 肌色優先設定 ──────────────┐
│                              │
│ 肌色優先: [ON/OFF]          │
│ 重み:     [====○───] 3.0   │
│ 最低枠数: [▼ 4色]          │
│                              │
│ ※ONにすると肌色の          │
│  グラデーションが改善します  │
└──────────────────────────────┘
```

## 4. 機能B: 領域分離型減色

### 4.1 概要

画像を「肌色領域」と「非肌色領域」に分離し、それぞれ独立にパレットを割り当てる。機能Aの重み付けより強力なアプローチ。

### 4.2 処理フロー

```
入力画像
  ↓
[Step 1] 肌色マスク生成
  ↓
[Step 2] 肌色領域と非肌色領域に分離
  ↓
[Step 3] パレット枠の配分決定
  ↓
[Step 4] 各領域で独立にメディアンカット
  ↓
[Step 5] 統合パレット（16色）を生成
  ↓
ディザリングへ
```

### 4.3 Step 1: 肌色マスク生成

- 機能Aと同じHSV+RGB条件で肌色判定
- 判定結果を2値マスク画像として保持
- **モルフォロジー処理**で平滑化:
  - 膨張（dilate）3×3カーネル × 2回: 小さな穴を埋める
  - 収縮（erode）3×3カーネル × 1回: 輪郭を整える
  - 目的: 肌色領域のノイズ除去と連結性の確保

### 4.4 Step 2: 領域分離

- マスク=1のピクセル → 肌色ピクセル配列へ
- マスク=0のピクセル → 非肌色ピクセル配列へ

### 4.5 Step 3: パレット枠配分

配分方式を選択可能:

```
パラメータ:
  split_mode: "auto" | "manual"

auto（デフォルト）:
  肌色ピクセル比率に応じて配分、ただし肌色側に有利な補正を入れる
  skin_ratio = 肌色ピクセル数 / 総ピクセル数
  skin_palette_count = clamp(
    round(skin_ratio * 16 * 1.5),  // 1.5倍の補正
    min: 4,
    max: 10
  )
  non_skin_palette_count = 16 - skin_palette_count

manual:
  UIスライダーで直接指定
  skin_palette_count: 4〜12（デフォルト: 8）
```

### 4.6 Step 4: 独立メディアンカット

- 肌色ピクセル配列に対して skin_palette_count 色のメディアンカット
- 非肌色ピクセル配列に対して non_skin_palette_count 色のメディアンカット
- 各結果を4096色グリッドにスナップ

### 4.7 Step 5: 統合パレット生成

- 肌色パレット + 非肌色パレット を結合して16色パレットとする
- 結合後、パレット内の色距離が近すぎるペア（CIE76色差 ΔE < 5）がある場合:
  - 片方を微調整して色差を確保

### 4.8 UI追加

パラメータパネルに以下を追加:

```
┌─ 領域分離設定 ──────────────┐
│                              │
│ 領域分離: [ON/OFF]          │
│ 配分方式: [▼ 自動]         │
│ 肌色枠数: [====○───] 8色   │
│                              │
│ [マスクプレビュー]          │
│  → 肌色検出結果を          │
│    オーバーレイ表示          │
└──────────────────────────────┘
```

マスクプレビューボタン押下時:
- メインプレビューに肌色検出マスクを赤色半透明でオーバーレイ表示
- ユーザーが検出精度を確認し、HSVパラメータを微調整できる

### 4.9 肌色検出パラメータの詳細設定（上級者向け）

メニュー「パレット(P)」→「肌色検出設定」で98風ダイアログを開く:

```

│                                      │
│ ── HSV範囲 ──                       │
│ H min: [====○──] 0°                │
│ H max: [======○] 50°               │
│ S min: [=○─────] 0.10              │
│ S max: [=====○─] 0.70              │
│ V min: [=○─────] 0.20              │
│ V max: [======○] 0.95              │
│                                      │

│                                      │
│         [ OK ]  [Cancel]             │
└──────────────────────────────────────┘
```

## 5. 機能C: 肌専用ディザパターン

### 5.1 概要

肌色領域のディザリングに特化したパターンを追加。通常の市松パターンは均一な分布だが、肌の場合はより滑らかに見えるパターンが有効。

### 5.2 肌用パターン定義

#### パターン1: スムースグラデーション（smooth_gradient）

肌色の明暗遷移で使用する非対称パターン:

```
4×4マトリクス:
  [  1,  9,  3, 11 ]
  [ 13,  5, 15,  7 ]
  [  4, 12,  2, 10 ]
  [ 16,  8, 14,  6 ]

特徴: 標準Bayer行列を肌色用に最適化
  - 隣接ピクセルの色差を最小化する配置
  - 斜め方向のパターン感を軽減
```

#### パターン2: ハーフトーン風（halftone）

印刷物のハーフトーンに近い円形配置:

```
6×6マトリクス:
  [ 35, 26, 22, 23, 27, 36 ]
  [ 25, 14,  9, 10, 15, 28 ]
  [ 21,  8,  1,  2, 11, 24 ]
  [ 20,  7,  4,  3, 12, 29 ]
  [ 30, 13,  6,  5, 16, 33 ]
  [ 34, 31, 19, 18, 32, 35 ]

特徴: 中心から外側に向かって閾値が上がる
  - 低濃度で小さなドット、高濃度で大きなドット
  - 肌の柔らかさを表現しやすい
```

#### パターン3: 斜め45°パターン（diagonal）

98時代に一部の絵師が使っていた斜線ベースのディザ:

```
4×4マトリクス:
  [  1,  5,  2,  6 ]
  [  9, 13, 10, 14 ]
  [  3,  7,  4,  8 ]
  [ 11, 15, 12, 16 ]

特徴: 斜め45°方向にドットが整列
  - 水平・垂直方向のパターン感を抑制
  - 98の水平走査線と干渉しにくい
```

### 5.3 適用ロジック

```
ディザリング処理時:
  1. 対象ピクセルが肌色マスク内か判定
  2. 肌色マスク内の場合:
     → 肌専用ディザパターンを使用
  3. 肌色マスク外の場合:
     → 通常選択されたディザリング手法を使用
```

- 肌色マスクは機能Bで生成したものを共用
- 機能Bが OFF の場合でも、肌専用ディザ用にマスクは内部生成する

### 5.4 境界処理

肌色領域と非肌色領域の境界でパターンの不連続が目立たないよう、境界にブレンドゾーンを設ける:

```
パラメータ:
  blend_width: 2〜8px（デフォルト: 4px）

処理:
  境界から blend_width ピクセル以内の領域:
    blend_ratio = 距離 / blend_width
    threshold = skin_threshold * (1 - blend_ratio)
                + normal_threshold * blend_ratio
```

### 5.5 UI追加

ディザリング設定に追加:

```
┌─ ディザリング設定 ──────────────────┐
│                                      │
│ 手法:     [▼ 市松パターン]         │
│ 強度:     [====○───] 80%           │
│                                      │
│ ── 肌専用ディザ ──                  │
│ 有効:     [ON/OFF]                  │
│ パターン: [▼ スムース]             │
│ ブレンド: [==○─────] 4px           │
└──────────────────────────────────────┘
```

## 7. 機能間の連携

### 7.1 動作モード

4つの機能は独立にON/OFF可能で、以下の組み合わせを想定:

```
推奨組み合わせ:

[初心者向け] プリセット（エロゲ標準）のみ
  → D:ON, A:OFF, B:OFF, C:OFF
  → プリセット選ぶだけで98エロゲ風

[標準]    肌色優先 + 肌専用ディザ
  → A:ON, B:OFF, C:ON, D:OFF
  → 自動減色ベースで肌色優先

[こだわり] 領域分離 + 肌専用ディザ
  → A:OFF, B:ON, C:ON, D:OFF
  → 最高品質（処理やや重い）

[全部盛り] 領域分離 + 肌専用ディザ + プリセットベース
  → B:ON, C:ON, D:ON (プリセットを初期値にして領域分離で微調整)
```

### 7.2 排他制御

- 機能A（肌色優先パレット）と機能B（領域分離型減色）は**排他**
  - 両方ONの場合、機能B（より高品質）を優先し、機能Aは無視
  - UIで一方をONにすると他方は自動OFFになる
- 機能D（プリセット）と機能A/Bの組み合わせ:
  - プリセット選択時はA/Bの自動減色は無効化される
  - ただし「プリセットをベースに自動微調整」オプションを用意:
    - プリセットの肌色パレットを固定し、残りの非肌色枠のみ自動減色

### 7.3 肌色マスクの共有

- 機能B/Cで使用する肌色マスクは1回だけ生成し、キャッシュして共用
- 肌色検出パラメータが変更された場合のみ再生成
- マスク生成はWeb Workerで実行

## 9. パフォーマンス考慮

### 9.1 肌色検出

- 640×400（256,000ピクセル）のHSV変換＋判定: 100ms以内目標
- モルフォロジー処理: 50ms以内目標
- 合計: 150ms以内

### 9.2 領域分離メディアンカット

- ピクセル分離: 50ms以内
- 各領域のメディアンカット: 通常の1.5倍程度（領域が小さいため相殺）
- 合計: 通常メディアンカットの2倍以内

### 9.3 マスクキャッシュ

- 肌色検出パラメータまたは入力画像が変更されない限り、マスクを再利用
- キャッシュはUint8Array（1byte/pixel）で保持: 640×400 = 250KB程度
