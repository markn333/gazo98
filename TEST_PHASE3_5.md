## T-03: 肌色検出パラメータ調整

**前提**: 画像Aをロード＆クロップ済み

| 2 | H max スライダーを左に動かし `30°` に設定 | スライダー横の値表示が `30°` に更新される | |

## T-04: 機能A — 肌色優先パレット

**前提**: 画像Aをロード＆クロップ済み。まずベースラインとして通常パレットを生成しておく

| # | 操作 | 確認 | 結果 |
|---|------|------|------|
| 1 | メニュー「パレット(P)」→「自動生成」で通常パレットを生成 → `[ 変換実行 ]` | 変換結果をベースラインとして目視記憶（特に肌の色段差に注目） | |
| 2 | モード選択を `肌色優先` に変更 | 「重み」スライダー（3.0）と「最低枠」セレクト（4色）が表示される。ステータスバーに「肌色優先パレット生成中...」→「パレット自動生成完了」と表示される。パレットグリッドの色が更新される | |
| 3 | パレットグリッド（16色）の肌色系の色を数える | オレンジ〜ベージュ〜ピンク系の色が4色以上ある（#1のベースラインより多い） | |
| 4 | `[ 変換実行 ]` をクリック | 変換結果の肌部分が#1のベースラインより滑らか（色の段差が少ない）。背景は#1とほぼ同等か若干粗い | |
| 5 | 「重み」スライダーを `5.0` に変更 → モードを `OFF` → `肌色優先` に戻す（パレット再生成をトリガー） | パレットグリッドが更新される。#3の時より肌色系の色がさらに多い | |
| 6 | 「最低枠」を `8色` に変更 → モードを `OFF` → `肌色優先` に戻す | パレットグリッド16色中、肌色系（暖色のオレンジ〜ベージュ系）が8色以上ある | |
| 7 | `[ 変換実行 ]` をクリック | 肌のグラデーションが#4よりさらに細かい。ただし背景は色数が減り粗くなっている | |

## T-05: 機能B — 領域分離型減色

**前提**: 画像Aをロード＆クロップ済み

| # | 操作 | 確認 | 結果 |
|---|------|------|------|
| 1 | モード選択を `領域分離` に変更 | 「配分」セレクト（自動）が表示される。#T-04で表示されていた「重み」「最低枠」は非表示。ステータスバーに「領域分離パレット生成中...」→「パレット自動生成完了」。パレットグリッド更新 | |
| 2 | パレットグリッドを確認 | 肌色系の色と非肌色系の色が明確に分かれて含まれている（肌色系が画像の肌比率×1.5倍程度の枠数） | |
| 3 | `[ 変換実行 ]` をクリック | 肌部分と背景部分の両方がそれなりの品質で変換されている | |
| 4 | 「配分」を `手動` に変更 | 「肌枠」スライダー（初期値8色、範囲4〜12）が出現する | |
| 5 | 「肌枠」スライダーを `10` に変更。スライダー横の表示を確認 | `10色` と表示されている | |
| 6 | モードを `OFF` → `領域分離` に戻す（再生成トリガー） → `[ 変換実行 ]` | 肌のグラデーションが#3より細かい。背景は16-10=6色しかないため#3より粗い | |
| 7 | 「肌枠」スライダーを `4` に変更 → `OFF` → `領域分離` → `[ 変換実行 ]` | 肌が#6より粗く、背景が#6より細かい（肌4色・非肌12色の配分） | |
| 8 | モードを `肌色優先` に切替 | 「配分」「肌枠」が消え、「重み」「最低枠」が表示される（排他動作） | |

## T-06: 機能C — 肌専用ディザパターン

**前提**: 画像Aをロード＆クロップ済み。パレット生成済み（プリセット「エロゲ標準」推奨）

| # | 操作 | 確認 | 結果 |
|---|------|------|------|
| 1 | 肌専用ディザ OFF の状態で `[ 変換実行 ]` | ベタ減色の変換結果を確認。肌部分に色の境界線（ベタ塗りの段差）が見える。これをベースラインとして記憶 | |
| 2 | 肌専用ディザのチェックボックスをON | 「パターン」セレクト（スムース）と「ブレンド」スライダー（4px）が表示される | |
| 3 | パターン `スムース`、ブレンド `4px` のまま `[ 変換実行 ]` | 肌部分: #1のベタ塗り段差がドットパターンに置き換わり、2色が交互に並んでグラデーションを表現。非肌部分（背景・服・髪）: #1と同じベタ減色のまま変化なし | |
| 4 | パターンを `ハーフトーン` に変更 → `[ 変換実行 ]` | 肌部分のドット配置が#3と異なる。ドットが中心から広がる円形配置になり、低濃度部で小さなドット群、高濃度部で大きな塗り潰しになる | |
| 5 | パターンを `斜め45°` に変更 → `[ 変換実行 ]` | 肌部分のドットが斜め45°方向に整列した縞模様になる。#3（不規則）#4（円形）とは明確に異なるパターン | |
| 6 | ブレンドを `2px` に変更 → `[ 変換実行 ]` | 肌色領域と非肌色領域の境界部分を目視確認。ディザパターンがある領域とベタ塗り領域の切り替わりが急で、境界線が目立つ | |
| 7 | ブレンドを `8px` に変更 → `[ 変換実行 ]` | #6と同じ境界部分を確認。ディザパターンが徐々に薄くなりベタ塗りに遷移している。#6より境界が目立たない | |
| 8 | 肌専用ディザのチェックボックスをOFF → `[ 変換実行 ]` | #1と同じベタ減色に戻る。肌部分のドットパターンが完全に消えている | |

## T-08: 機能間連携・排他制御

**前提**: 画像Aをロード＆クロップ済み

| # | 操作 | 確認 | 結果 |
|---|------|------|------|
| 1 | モード `肌色優先` を選択 | 「重み」「最低枠」が表示される。「配分」「肌枠」は表示されていない | |
| 2 | モード `領域分離` に変更 | 「重み」「最低枠」が消え、「配分」が表示される。2つの設定が同時に表示されることはない | |
| 3 | モード `OFF` に変更 | 「配分」が消え、モード選択のみの表示。追加設定は何も表示されていない | |
| 4 | モード `領域分離` に設定＋肌専用ディザ ON＋パターン `スムース` → `[ 変換実行 ]` | 変換結果の肌部分にディザパターンが適用されている（ドットの交互配置が見える）。かつ、パレットが領域分離で生成されたもの（肌色系が多い）が使われている | |
| 5 | モード `OFF`＋肌専用ディザ ON → `[ 変換実行 ]` | 肌部分にディザパターンが適用されている（モードOFFでも肌専用ディザは独立して動作する）。パレットは直前に設定されたものがそのまま使われる | |
| 6 | プリセット「エロゲ標準」を選択 → 肌専用ディザ ON → `[ 変換実行 ]` | エロゲ標準のパレット色で変換され、肌部分にはディザパターンが適用されている | |
| 7 | メニュー「ファイル」→「閉じる」 | ドロップゾーンに戻る。肌感改善パネルを確認: モードが `OFF` でグレーアウト。肌専用ディザがOFFでグレーアウト。`[ マスク確認 ]` `[ 検出設定 ]` がグレーアウト。全コントロールがクリック不可 | |
